# Versioning

JobWize follows **Semantic Versioning (SemVer)**.

Version format:

```text
MAJOR.MINOR.PATCH
```

Example:

```text
1.0.0
```

Where:

- **MAJOR** → Breaking changes
- **MINOR** → New features (backward compatible)
- **PATCH** → Bug fixes and small improvements

Examples:

```text
1.0.0

1.1.0

1.1.1

2.0.0
```

# Release Process

Every release should:

1. Update the version number.
2. Move items from **Unreleased** into the new version.
3. Publish release notes.
4. Create a Git tag.
5. Update deployment pipelines if required.
